## Impresi Pertama

Awalnya, saya termasuk orang yang ragu menggunakan Chakra UI atau [TailwindCSS]() yang menjadi basis dasar / fondasi si Chakra UI ini. Karena menurut saya terlalu berat untuk saya yang biasa menggunakan _component library_ yang mempunyai banyak contoh-contoh _layout_ yang bisa langsung digunakan. Seperti [Material UI]() misalnya, mereka punya banyak komponen-komponen _layout_ yang sudah jadi dan dapat langsung digunakan. Seperti `AppBar / Navbar`, `Sidebar`, `Notification`, `Toast`, dan masih banyak lagi.

Namun, setelah saya memberi waktu untuk mencoba Chakra UI ini ternyata memberi kesan yang berbeda untuk saya. Saya akan jelaskan kelebihan & kekurangan nya (menurut pandangan pribadi saya) di bawah ini.

### Kelebihan

- Kelebihan yang paling utama dan membuat beberapa poin kelemahan di bawah seakan bisa ditoleransi adalah, fitur aksesibilitas (_accessibility_) built-in! Hal ini membuat saya tidak lagi perlu khawatir website saya tidak bisa diakses oleh sebagian orang karena keterbatasan "akses".

- **Flexibilitas tinggi!** Chakra UI ini mempunyai beberapa komponen yang sudah siap dipakai seperti `Button`, `Input`, `Radio`, `Progress`, dan lain-lain. Tapi karena Chakra UI ini berbasis TailwindCSS maka kita dapat memodifikasi komponen-komponen yang sudah tersedia sesuai dengan keinginan kita.

- **Dark mode!** Berbeda dengan TailwindCSS yang baru-baru ini mengeluarkan fitur dark mode, Chakra UI sudah lebih dahulu memilikinya. Cara menggunakan nya pun sangat mudah. Sudah tersedia hooks yang dapat kita gunakan yaitu `useColorMode`. Lihat selengkapnya, [di sini](https://chakra-ui.com/docs/features/color-mode#usecolormode).

- **Learning curve** nya mudah dan intuitif. Karena Chakra UI mengadopsi _prop based model styling_ yang sudah digunakan oleh _component library_ yang lain di luar sana, seperti Material UI, [Ant Design](https://ant.design/), dan lain-lain.

- **Array & object syntax API!** Ini gokil banget sih. Kita bisa dengan mudah melakukan _responsive styling_ pada komponen dengan memberikan _attribute_ atau _props_ yang berisi array & object. Ini membantu saya supaya tidak perlu repot-repot menulis `@media (min-width) ...` di dalam file CSS. Chakra UI sudah menyediakan 4 _responsive breakpoint_ yang bisa kita gunakan. Lihat selengkapnya, [di sini](https://chakra-ui.com/docs/theming/theme#breakpoints).

### Kelemahan

- Saya sepertinya melakukan kesalahan juga dalam membangun arsitektur blog ini menggunakan Chakra UI + MDX. Karena saat saya mencoba me-_render_ halaman blog, muncul sebuah _warning_ karena apa yang di-_render_ dari _server-side_ berbeda dengan _client-side_ nya. Yang membuat berbeda adalah saat di-_render_ di _server-side_ CSS dari Chakra UI berbentuk _class name_ saja. Tapi setelah masuk ke _client-side, class name_ tersebut berubah menjadi _inline styles_. Nah di sini saya belum mencari tau apakah dari sisi Chakra UI atau dari MDX nya yang bermasalah. Saat ini saya hanya menemukan _workaround_ yang _unofficial_, yaitu dengan me-_render_ halaman blog ini di _client-side_ saja.

- Saya tidak biasa menulis `margin`, `padding`, `width`, `height` di _props_. Jadi terkadang saya harus berhenti, dan membaca dokumentasi untuk mencari tau seperti apa saya harus menulisnya.

- Tidak adanya komponen-komponen penting dalam pembuatan _layout_, seperti `AppBar / Navbar`, `Sidebar`, dan lain-lain. Jadi kita harus membuatnya sendiri. Dan mungkin akan menjadi pekerjaan yang berat jika dihadapkan dengan _deadline_ yang sangat mepet.

Ternyata Chakra UI sangat bagus, dan saya sangat senang menyediakan waktu untuk mempelajarinya. Saya puas dengan hasil yang diberikan oleh _component library_ yang satu ini.
